[2026-06-10] v0.2.2: Lenient LLM output parsing across all agents — GBNF grammars are llama.cpp-only; Ollama ignores them, so strict serde parsing failed on real model output and the detector returned "Silent (parse error)" on every run for Ollama users. Principle: prompts ask for the strict shape, parsers accept the sloppy reality, fallback stays Silent.

[2026-06-11] v0.2.2 AFK: idle_start is backdated to actual input stop (was detection time, +5min error per break); the open app_focus duration is capped at idle onset and a fresh focus event opens after idle_end (durations previously sailed through breaks — a 710-minute "browsing" event was a night's sleep); the away length lands on the idle_start row so the UI shows "Away · 42m". Media playback suppresses idle via IOPMCopyAssertionsStatus (display-sleep assertions) — watching a video is not AFK, and it's exactly the activity drift detection must keep seeing. Reads assertion counts only.
[2026-06-11] v0.2.2 sleep detection: system sleep freezes timers, so the idle check can never fire across a closed lid — the cause of multi-hour phantom durations. A poll-to-poll wall-clock jump > 60s emits a synthetic away period covering the gap. Both platforms. Historical pre-fix durations are left as-is (no in-span signal exists to repair them honestly); they age out via the 14-day prune.
[2026-06-11] v0.2.2 detail accuracy: summarize prompts number events 1..N per chunk but were resolved against the global event list — every chunk after the first wrote wrong per-event descriptions (the "Clash Party – using iTerm2" mush). Now chunk-relative. Vision/OCR screenshots fire 2s after a focus switch and attach to the event that just STARTED (was: attributed to the outgoing event whose window was already gone). Prompt no longer claims events are newest-first (they're oldest-first). LLM calls moved outside the persistence transaction — a write lock held across minute-long awaits would have blocked the capture loop.
[2026-06-11] v0.2.2 sessions: idle breaks and >15min gaps are hard segment boundaries for the summarizer (split_at_breaks, unit-tested) — sessions never span a break, killing the 9-hour mega-session failure mode deterministically rather than by prompt engineering.
